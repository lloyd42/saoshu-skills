# 新手读报告指南

这份说明不是讲内部实现，而是告诉第一次使用 `saoshu` 的用户：

- 报告应该先看哪里
- 哪些信息最值得相信
- 哪些地方代表“还不能下结论”

如果你只想快速决定“这本书要不要继续看”，按下面顺序看就够了。

## 1. 先看「一眼结论」

这是最省时间的入口。

重点看：

- `结论`
- `推荐指数`
- `风险等级`
- `置信度`
- `一句话`

理解方式：

- `可看`：当前覆盖下没看到会直接劝退的硬风险
- `慎入`：已经有明显风险或不确定点，适合谨慎用户停一下
- `劝退`：已经出现高危信号或关键事件，继续投入时间的性价比很低

## 2. 再看「为什么这样判断」

如果你想知道这个结论是不是靠谱，第二步只看两块：

- `关键已确认事件`
- `最重要的未证实风险`

这一步的意义是：

- 已确认事件告诉你：系统已经找到哪些足以影响结论的关键情节
- 未证实风险告诉你：还有哪些点可能会推翻现在的判断

如果这里已经出现你明确不能接受的内容，就不需要继续往下看了。

## 3. 最后只补 3 个最关键问题

如果你仍然犹豫，不要自己重新通读整本书。

优先看报告里的：

- `如果还不确定，先补这3个问题`

这 3 个问题会优先挑：

- 最可能改变结论的风险
- 六大高危风险相关问题
- 已经有较强迹象、但还差最后一锤的点

正常用户只需要补这 3 个问题，不需要把所有 `待补证` 全都处理一遍。

## 4. 什么时候再切到专家视图

只有当你满足下面任一情况时，再切到 `expert`：

- 你准备继续追证
- 你想看完整事件链
- 你想确认抽样覆盖是否足够
- 你想借助关系图理解长书人物结构

专家视图会展开：

- 事件候选复核表
- 雷点/郁闷点表
- 术语速查
- 抽样依据与审计面板

## 5. 什么情况不要过度解读

以下内容代表“还不能拍板”：

- `未证实风险`
- `待补证`
- 低覆盖率下的 economy 结论

它们的正确用法是：

- 先帮助你知道要不要继续查
- 不是让你把它们直接当作实锤结论

## 6. 建议阅读顺序

对大多数用户，推荐始终按这个顺序：

1. `一眼结论`
2. `为什么这样判断`
3. `如果还不确定，先补这3个问题`
4. 仍然不放心，再看 `深入查看`
